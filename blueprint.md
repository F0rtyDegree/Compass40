### **План реализации функции "Интерактивная Карта"**

**Цель:** Создать в приложении новый раздел, где пользователь может загрузить изображение спортивной карты, откалибровать его и расставить контрольные пункты (КП) для автоматического расчета дистанции и азимута между ними.

**Шаг 0: Подготовка и создание структуры**
1.  **Обновить `blueprint.md`:** Внести в документ этот план, чтобы зафиксировать нашу цель и шаги. (Выполнено)
2.  **Добавить зависимость:** Добавить в `pubspec.yaml` пакет `image_picker` для выбора изображений из галереи.
3.  **Создать новый экран:** Создать новый файл `lib/map_planner_screen.dart` для будущего экрана планировщика.
4.  **Добавить навигацию:** На главном экране (`lib/main.dart`) добавить новую кнопку (например, иконку карты), которая будет вести на `MapPlannerScreen`.

**Шаг 1: Загрузка и отображение карты**
1.  На экране `MapPlannerScreen` создать кнопку "Загрузить карту".
2.  Используя `image_picker`, реализовать логику выбора изображения из галереи телефона.
3.  Отобразить выбранное изображение на весь экран.
4.  *Проверка:* Мы должны убедиться, что пользователь может выбрать картинку и увидеть ее в приложении.

**Шаг 2: Расстановка маркеров КП**
1.  Использовать виджет `GestureDetector` поверх изображения карты, чтобы отлавливать нажатия пользователя.
2.  При каждом нажатии сохранять координаты `(x, y)` точки в список.
3.  В месте каждого нажатия отображать на карте визуальный маркер (например, красный круг с номером КП).
4.  *Проверка:* Мы должны иметь возможность "натыкать" несколько точек на карте, и они должны остаться на своих местах.

**Шаг 3: Калибровка масштаба и расчет дистанции**
1.  Создать режим "Калибровка масштаба".
2.  Реализовать интерфейс, где приложение просит пользователя:
    а. Тапнуть на первую точку для калибровки.
    б. Тапнуть на вторую точку.
    в. Ввести реальное расстояние в метрах между этими точками в текстовое поле.
3.  На основе этих данных рассчитать и сохранить коэффициент масштаба (`пиксели/метр`).
4.  Реализовать логику расчета дистанции между двумя последними поставленными КП, используя этот коэффициент.
5.  Вывести на экран рассчитанную дистанцию.
6.  *Проверка:* После калибровки и расстановки двух КП, приложение должно показывать осмысленное расстояние в метрах.

**ШаГ 4: Калибровка севера и расчет азимута**
1.  Создать режим "Калибровка севера".
2.  Реализовать интерфейс, где приложение просит пользователя "провести линию, указывающую на север" (тапнуть на начало и конец линии севера на карте).
3.  Рассчитать и сохранить угол отклонения севера от вертикальной оси изображения.
4.  Реализовать логику расчета азимута на следующий КП, учитывая этот угол отклонения.
5.  Вывести на экран рассчитанный азимут в градусах.
6.  *Проверка:* Приложение должно показывать азимут, который меняется в зависимости от расположения КП.

**Шаг 5: Интеграция и завершение**
1.  Передать рассчитанные азимут и дистанцию с экрана планировщика на главный экран компаса, чтобы они использовались как целевые значения.
2.  Добавить кнопки для сброса/удаления КП и начала планирования заново.
3.  Добавить простые текстовые подсказки для пользователя на каждом этапе.
